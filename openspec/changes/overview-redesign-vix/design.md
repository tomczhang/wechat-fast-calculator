## Context

当前总览页是扁平列表，VOO/QQQ 和七姐妹混排，没有信息层级。用户核心需求是「宽指跌到什么位置了？要不要加仓？」，需要一眼看到结论。

## Goals / Non-Goals

**Goals:**
- 总览页三层信息架构：宽指置顶 → VIX 恐慌指标 → 个股列表
- 击球点（Strike Zone）视觉标记，让用户一眼判断是否触发加仓
- VIX 实时数据获取与恐慌等级展示

**Non-Goals:**
- 不做自动提醒/推送通知
- 不改变加仓金额计算逻辑
- 不删除宽指/个股独立 tab 页

## Decisions

### 1. VIX 数据源：MASSIVE API `I:VIX`
MASSIVE 的 indices ticker 格式为 `I:VIX`，返回日 K 数据。与现有 ticker 共享同一套 getBars + 缓存逻辑。
服务端新增 `/api/vix` 端点，返回当前值、涨跌幅、恐慌等级。

### 2. 击球点阈值硬编码在前端
阈值（VOO -6.5%、QQQ -9%、个股 -30%）是用户个人策略，变动频率低。
硬编码在小程序前端，避免增加服务端复杂度。后续可改为可配置。

### 3. 总览页布局重构，保留 tab 页
总览成为信息聚合主页，宽指/个股 tab 保留详细信息。
总览只展示关键指标 + 击球点状态，不展示前高日期等细节。

### 4. VIX 恐慌等级纯前端判定
三个阈值（30/40/50）在前端判定并映射颜色和标签，服务端只返回原始数值。

## Risks / Trade-offs

- [VIX 非交易品] MASSIVE 免费 tier 对 indices 的支持可能与股票不同 → 先测试 `I:VIX` 是否返回数据，失败则用 `VIXY` ETF 作为替代
- [击球点硬编码] 用户策略变化需改代码 → 阈值集中定义为常量，后续可改为服务端配置
