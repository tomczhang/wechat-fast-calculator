## Context

本项目是一个微信小程序，用于计算美股宽指（VOO + QQQ）的组合跌幅，辅助判断加仓时机。数据来源为 MASSIVE API（与 patient-hunter 项目共用 API Key）。小程序需要云函数来保护 API Key 并实现请求缓存。

## Goals / Non-Goals

**Goals:**
- 提供 VOO/QQQ 近6个月最高价和当前价
- 计算 70/30 加权组合跌幅百分比
- 30s 内存缓存避免超出 MASSIVE 免费账户速率限制（5次/分钟）
- 简洁单页展示，手动刷新

**Non-Goals:**
- 不做用户认证/登录
- 不做历史数据存储或趋势图表
- 不做自动推送通知
- 不支持自定义标的或权重调整

## Decisions

### 1. 架构：微信云开发（CloudBase）

**选择**：使用微信云开发的云函数作为后端，而非自建服务器。

**理由**：
- 云函数免费额度足够（每月 10 万次调用）
- 无需额外购买服务器和域名备案
- 与小程序天然集成，调用简单（`wx.cloud.callFunction`）

**替代方案**：自建 Node.js 后端 → 需要服务器、域名备案、HTTPS证书，成本和复杂度高

### 2. 缓存策略：云函数内存级缓存

**选择**：云函数内使用模块级变量缓存，30s TTL。

**理由**：
- MASSIVE 免费账户限制 5 次/分钟，每次计算需要 2 次 API 调用（VOO + QQQ），30s 缓存确保安全
- 云函数实例有复用机制，内存缓存在实例存活期间有效
- 实现简单，无需引入 Redis 等外部依赖

**缓存结构**：`{ data, timestamp }` 格式，请求时检查是否在 30s 内，是则直接返回

### 3. 计算方式：按金额加权（非百分比加权）

**选择**：`前高总额 = 0.7 × VOO_high + 0.3 × QQQ_high`，`当前总额 = 0.7 × VOO_current + 0.3 × QQQ_current`

**理由**：模拟实际持仓比例，直接反映组合市值变化

### 4. QQQ 而非 QQQM

**选择**：使用 QQQ（Invesco QQQ Trust）而非 QQQM。

**理由**：QQQ 流动性更好、知名度更高，是纳指100最主流的 ETF。

## Risks / Trade-offs

- **[云函数冷启动]** → 首次调用可能有 1-2s 延迟，可接受。缓存命中后响应很快。
- **[MASSIVE API 不可用]** → 前端展示错误提示，用户可手动重试。
- **[缓存失效粒度]** → 云函数实例回收后缓存丢失，但会在下次调用时自动重建。
- **[30s 缓存导致数据延迟]** → 对于判断加仓时机场景，30s 延迟完全可接受。
